# せどりレーダー デプロイ手順（Vercel + Turso）

このサイトを一般公開するための手順です。**アカウント作成・認証キーの取得・実際のデプロイは本人操作が必要**なため、この手順に沿って進めてください。所要時間の目安は 30〜40分。

構成のおさらい:
- **Vercel**: サイト本体（Next.js）をホスティング。
- **Turso**: 本番データベース（SQLite互換のクラウドDB）。
- **スクレイピング**: これまで通り**ローカルPCのタスクスケジューラ**で実行し、結果をTursoに書き込む。VercelのサイトはそのTursoを読むだけ。

---

## 全体の流れ

```
[ローカルPC] --scrape--> [Turso(クラウドDB)] <--読み取り-- [Vercel(公開サイト)] <-- 閲覧者
```

---

## STEP 1: Turso のセットアップ（DB作成）

1. https://turso.tech/ にアクセスし、GitHubアカウント等でサインアップ（無料プランでOK）。
2. Turso CLI をインストール（PowerShell）:
   ```powershell
   # 方法A: 公式スクリプト（WSL/Git Bash推奨）。うまくいかない場合は方法B。
   # 方法B: scoop を使う場合
   scoop install turso
   ```
   ※ Windowsでうまく入らない場合は、Tursoのダッシュボード（Web UI）上でDB作成・トークン発行まで完結できます（CLI不要）。
3. ログイン: `turso auth login`
4. データベースを作成（東京リージョン `nrt` を指定）:
   ```powershell
   turso db create sedori-radar --location nrt
   ```
5. **接続URL** を取得（`libsql://sedori-radar-xxxx.turso.io` の形式）:
   ```powershell
   turso db show sedori-radar --url
   ```
6. **認証トークン** を発行:
   ```powershell
   turso db tokens create sedori-radar
   ```
   → 表示された URL とトークンを控えておく（STEP 2・4で使う）。

---

## STEP 2: Turso にテーブルを作成（スキーマ適用）

このリポジトリの `prisma/schema_turso.sql` を Turso に流し込みます。

```powershell
# CLIがある場合
Get-Content prisma/schema_turso.sql | turso db shell sedori-radar
```

CLIが使えない場合は、Tursoダッシュボードの「SQL console」（DBを選んで開く）に `prisma/schema_turso.sql` の中身を貼り付けて実行してもOKです。

> スキーマを変更したとき（`prisma/schema.prisma` を編集したとき）は、
> `npx prisma migrate diff --from-empty --to-schema prisma/schema.prisma --script -o prisma/schema_turso.sql`
> で `schema_turso.sql` を作り直し、差分を Turso に適用してください。

---

## STEP 3: ローカルからTursoへ初回データ投入

ローカルの `.env` に Turso の接続情報を追記します（`.env.example` を参照）:

```
TURSO_DATABASE_URL="libsql://sedori-radar-xxxx.turso.io"
TURSO_AUTH_TOKEN="（STEP1で発行したトークン）"
```

この状態でスクレイピングを実行すると、**ローカルではなくTursoに書き込まれます**:

```powershell
npm run scrape
```

- プレバンは1回120件までの差分取得なので、初回は全件そろえるため `npm run scrape` を2〜3回実行してください。
- 以降、`run_scrape.bat`（タスクスケジューラ登録済みなら自動）もTursoに書き込みます。

> **注意**: `.env` に `TURSO_*` を設定している間は、ローカル開発サーバー（`npm run dev`）もTursoを見ます。
> ローカルのSQLiteで開発したいときは `.env` の `TURSO_*` を一時的にコメントアウトしてください。

## STEP 4: Vercel にデプロイ

### 4-1. コードをGitHubに上げる（推奨ルート）

`sedori_site` フォルダを1つのGitリポジトリとしてGitHubにpushします。
（`.gitignore` で `.env`・`node_modules`・ローカルDBは除外済み）

```powershell
cd C:\Users\user\Documents\kurodo\sedori_site
git init
git add .
git commit -m "せどりレーダー 初回公開"
# GitHubで空リポジトリを作成し、そのURLを origin に設定
git remote add origin https://github.com/<あなた>/sedori-radar.git
git branch -M main
git push -u origin main
```

### 4-2. Vercelでインポート

1. https://vercel.com/ にGitHubアカウントでサインアップ（無料 Hobbyプラン）。
2. 「Add New… → Project」→ 先ほどのリポジトリを選択。
3. Framework Preset は **Next.js** が自動選択される。Root Directory はリポジトリ直下（`sedori_site` をそのまま上げた場合は変更不要）。
4. **Environment Variables** に以下を登録:
   | Key | Value |
   |---|---|
   | `TURSO_DATABASE_URL` | `libsql://sedori-radar-xxxx.turso.io` |
   | `TURSO_AUTH_TOKEN` | （STEP1のトークン） |
   | `NEXT_PUBLIC_SITE_URL` | （デプロイ後のURL。最初は空でOK、後述） |
5. 「Deploy」を押す。ビルド（`prisma generate && next build`）が走り、数分で公開URLが発行されます。

### 4-3. サイトURLを環境変数へ反映（任意）

発行された `https://sedori-radar-xxxx.vercel.app` を、Vercelの環境変数 `NEXT_PUBLIC_SITE_URL` に設定して再デプロイすると、robots.txt / sitemap.xml / OGタグに正しいURLが入ります（SEO向け）。

---

## STEP 5: 公開後の運用

- **データ更新**: ローカルPCの `run_scrape.bat`（タスクスケジューラで毎日8:00/20:00）がTursoに書き込み → サイトは常に最新。PCが起動している時間に更新されます。
- **PCを起動していない時間も更新したい場合**: スクレイパーをクラウド（GitHub Actions等）で回す構成に切り替える必要があります（別途相談）。
- **独自ドメイン**: VercelのProject → Settings → Domains から追加できます。

---

## トラブルシューティング

- **Vercelのビルドで `prisma generate` が失敗する** → `package.json` の `build` が `prisma generate && next build`、`postinstall` が `prisma generate` になっているか確認。
- **サイトは表示されるがデータが0件** → Vercelの環境変数 `TURSO_DATABASE_URL`/`TURSO_AUTH_TOKEN` が正しいか、STEP3でTursoにデータが入っているか確認。
- **スクレイプがローカルDBに書き込まれてしまう** → `.env` の `TURSO_DATABASE_URL` が設定されているか確認（未設定だと `file:./dev.db` にフォールバック）。
